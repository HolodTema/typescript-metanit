## Введение в асинхронность

Асинхронное программирование - способ написания кода, когда задачи могут выполняться
одновременно, хотя на самом деле они на пуле потоков быстро чередуют друг друга. Но
для юзера эффект такой, что задачи выполняются асинхронно.

Весь код JS и TS работает в одном потоке. Если бы не было асинхронности, то при выполнении
длительной работы, например, при отправке запроса на сервер, UI бы зависал, дожидаясь ответа
от сервера.

### Event Loop

Event Loop - событийный цикл - механизм асинхронности в JS и TS.

Когда мы запускаем асинхронную операцию, она передается в WebAPI (если мы фронтенд) или в
Node.js-API (если мы на бэке) и выполняется вне основного потока.

По завершении колбэк асинхронной операции помещается в очередь задач.

Event loop следит за изменениями в очереди задач. Если есть новая задача, event loop берет
ее на выполнения в стек вызовов.

### Простой пример

```ts
console.log("A");

setTimeout(() => {
    console.log("B")
}, 0);

console.log("C");
```

Здесь функция внутри setTimeout() выполняется асинхронно. Хотя задержка ноль миллисекунд,
все равно выполнение будет таким:

```
A C B
```

То есть setTimeout() ждет 0мс и после этого асинхронно вызывает наш колбэк. То есть помещает
этот колбэк в очередь задач.

Event loop сначала выполнит синхронный код, а потом возьмет из очереди задач наш колбэк и
выполнит его.

## Файл конфигурации tsconfig.json

Нужен чтобы сконфигурировать проект на TS

- устанавливает корневой каталог проекта TS

- устанавливает настройки компилятора

- устанавливает файлы проекта

### Добавим tsconfig.json в корень проекта

```json
{
    "compilerOptions": {
        "target": "es5",
        "removeComments": true,
        "outDir": "js",
        "sourceMap": true
        "outFile": "main.js"
    },
    "exclude":[
        "wwwroot",
        "node_modules"
    ],
    "files":[
        "app.ts",
        "interfaces.ts",
        "classes.ts",
    ],
    "compileOnSave": true
}
```

### Раздел compileOptions

target - какая версия ES в проекте

removeComments - нужно ли конвертировать комментарии из ts в js

outDir - куда сохранять скомпилированные JS файлы

sourceMap - нужно ли генерировать словарь для сопоставления
.ts и скомпилированных .js файлов.

### Раздел exclude

какие файлы и папки исключить из компиляции

### Раздел files

Какие файлы включить в компиляцию.

Если раздел files не указан, будут включены все .ts файлы в папке и в подпапках проекта.

### compileOnSave

Если true, то компилятор будет компилить ts файлы при каждом сохранениии автоматом.

### Когда юзается tsconfig.json

tsconfig.json юзается компилятором, когда при вызове команды компилятора в bash
не указываются файлы. И тогда просто просматривается текущая папка, если находится
tsconfig.json - то компиляция идет по правилам из этого конфига.

```bash
tsc
# здесь файлов не передали, поэтому найдется tsconfig.json в текущей папке

tsc file.ts
# здесь скомпилируется 1 файл, tsconfig.json не сработает
```
